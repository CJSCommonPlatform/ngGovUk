Purpose

The purpose of this document is to provide UX team a playground, so they can quickly experiment with Frontend related stuff like HTML, CSS and JavaScript; can showcase/share their creation without having to worry about the following:

  • Setting up the Dev enviroment on thier machines.
  • Cloning and pulling the entire project from repositories.

Plunker

The tool we decided to create this playground name Plunker which offers the following advantages:

  1. User can create & edit an arbitrary number of files.
  2. Downlaod the project as a ZIP.
  3. Better feedback and realtime preview as you type.
  4. Mimic the project structure using file names like 'assets/my-file.extention'
